# Mate 웹사이트 발표 가이드

다른 사람에게 Mate 웹사이트를 설명할 때 사용할 수 있는 구조화된 가이드입니다.

---

## 🎯 30초 엘리베이터 피치

"Mate는 유니티 외주 개발과 개발 강의에 특화된 전문 팀의 포트폴리오 웹사이트입니다. Next.js 14와 TypeScript로 제작되었으며, 3D 기하학적 효과가 적용된 모던한 디자인과 완전한 반응형 레이아웃을 제공합니다."

---

## 📋 발표 구조 (5분 버전)

### 1. 프로젝트 소개 (30초)
- **이름**: Mate 외주개발팀 웹사이트
- **목적**: 팀의 역량과 서비스를 소개하는 포트폴리오 사이트
- **타겟**: 유니티 외주 개발 및 개발 강의를 찾는 클라이언트

### 2. 기술 스택 (1분)
- **Next.js 14**: 최신 App Router 사용
- **TypeScript**: 타입 안정성
- **Tailwind CSS**: 빠른 스타일링
- **Framer Motion**: 부드러운 애니메이션
- **정적 사이트 생성**: 빠른 로딩 속도

### 3. 주요 기능 (2분)

#### 3.1 랜딩 페이지
- **3D 기하학적 효과**
  - 부유하는 원형 도형
  - 3D 회전 큐브, 삼각형, 육각형
  - 애니메이션 라인과 파티클
- **반응형 디자인**: 모든 기기에서 완벽하게 작동

#### 3.2 페이지 구성
- **홈페이지**: 서비스, 프로젝트, 팀 역량 개요
- **서비스 페이지**: 4가지 주요 서비스 소개
- **팀 역량 페이지**: 통계와 역량 카드
- **프로젝트 포트폴리오**: 완료된 프로젝트 소개
- **연락처 페이지**: 문의 양식

#### 3.3 기술적 특징
- **이미지 예외 처리**: 이미지 로딩 실패 시 플레이스홀더
- **SEO 최적화**: 메타데이터, sitemap, robots.txt
- **타입 안정성**: TypeScript로 모든 데이터 타입 정의

### 4. 데이터 구조 (30초)
- **데이터 중심 설계**: `src/data/` 폴더에 모든 콘텐츠
- **쉬운 업데이트**: 데이터 파일만 수정하면 자동 반영
- **타입 안전성**: 인터페이스로 데이터 구조 보장

### 5. 배포 및 운영 (1분)
- **GitHub 연동**: 코드 버전 관리
- **Vercel 배포**: 자동 배포 및 SSL 인증서
- **커스텀 도메인**: 직접 구매한 도메인 연결 가능
- **정적 사이트**: 빠른 로딩, SEO 친화적

## 📊 상세 설명 구조 (15분 버전)

### 1. 프로젝트 배경 및 목적 (2분)

#### 왜 만들었나?
- 팀의 역량을 체계적으로 소개
- 외주 개발 및 강의 서비스 홍보
- 전문성과 신뢰도 구축

#### 타겟 고객
- 유니티 게임 개발이 필요한 클라이언트
- 개발 강의를 찾는 학습자
- 외주 개발을 고려하는 기업

### 2. 기술 선택 이유 (3분)

#### Next.js 14 선택 이유
- **App Router**: 최신 기능과 성능
- **정적 사이트 생성**: 빠른 로딩, SEO 최적화
- **타입스크립트 지원**: 타입 안정성
- **Vercel 최적화**: 배포 간편

#### Framer Motion 선택 이유
- **부드러운 애니메이션**: 사용자 경험 향상
- **성능 최적화**: GPU 가속
- **선언적 API**: 코드 가독성

#### Tailwind CSS 선택 이유
- **빠른 개발**: 유틸리티 클래스
- **일관된 디자인**: 디자인 시스템
- **번들 크기 최적화**: 사용한 클래스만 포함

### 3. 주요 기능 상세 설명 (5분)

#### 3.1 랜딩 페이지 Hero 섹션
**기하학적 효과 구현**
- CSS 3D Transform 활용
- Framer Motion으로 부드러운 애니메이션
- 여러 레이어로 입체감 표현

**성능 최적화**
- `transform` 속성 사용 (GPU 가속)
- `will-change` 속성으로 최적화
- 적절한 애니메이션 지속 시간

#### 3.2 팀 역량 페이지
**데이터 중심 설계**
- `teamCapabilities.ts`에 모든 역량 데이터
- 통계와 역량 카드로 시각화
- "왜 선택해야 할까요?" 섹션으로 신뢰도 구축

#### 3.3 프로젝트 포트폴리오
**동적 라우팅**
- `[slug]` 동적 라우트로 프로젝트 상세 페이지
- `generateStaticParams`로 빌드 시 모든 페이지 생성
- 프로젝트 데이터와 팀원 데이터 연결

#### 3.4 이미지 처리
**SafeImage 컴포넌트**
- 이미지 로딩 실패 시 플레이스홀더
- 로딩 상태 표시
- 에러 핸들링

### 4. 데이터 구조 및 관리 (2분)

#### 데이터 파일 구조
```
src/data/
├── team.ts              # 팀원 정보
├── teamCapabilities.ts  # 팀 역량
├── projects.ts         # 프로젝트 정보
└── services.ts         # 서비스 정보
```

#### 타입 정의
- TypeScript 인터페이스로 데이터 구조 정의
- 타입 안정성 보장
- IDE 자동완성 지원

#### 콘텐츠 업데이트 워크플로우
1. 데이터 파일 수정
2. `npm run build` 실행
3. Git 커밋 및 푸시
4. 자동 배포 (Vercel)

### 5. SEO 및 성능 최적화 (2분)

#### SEO 전략
- **메타데이터**: 각 페이지별 고유 title, description
- **키워드 최적화**: "유니티 외주 개발", "개발 강의" 등
- **구조화된 데이터**: 준비됨 (추가 구현 가능)
- **sitemap.xml**: 동적 생성
- **robots.txt**: 검색 엔진 크롤링 제어

#### 성능 최적화
- **정적 사이트 생성**: 빌드 시 모든 페이지 생성
- **이미지 최적화**: 적절한 해상도 사용
- **코드 분할**: Next.js 자동 분할
- **CDN**: Vercel 글로벌 CDN

### 6. 배포 및 운영 (1분)

#### 배포 플랫폼
- **Vercel**: GitHub 연동, 자동 배포
- **커스텀 도메인**: 직접 구매한 도메인 연결
- **SSL 인증서**: 자동 발급

#### 업데이트 프로세스
1. 로컬에서 개발 및 테스트
2. Git 커밋 및 푸시
3. Vercel 자동 배포
4. 도메인으로 접속 확인
